# PrivacyTap 威胁模型

## 保护资产

- Prompt、代码和工具参数中的业务敏感信息；
- Prompt 中误输入的认证凭证；
- 请求级占位符映射；
- 本地安全归档及可选观测输出。

## 信任边界

本机 PrivacyTap 进程与调用客户端属于信任域。第三方模型、外部观测系统、日志备份和普通运维访问者不应获得原始敏感值。

## 防御范围

- PII 在请求离开本机前替换；
- 当前请求认证 Key 在请求上游前精确阻断；
- 代码中的示例凭证使用可逆匿名化；
- Vault 不序列化、不落盘、不上传；
- exporter 只接收脱敏安全事件；
- Authorization Header 可转发但不记录；
- 并发请求使用独立 Vault；
- exporter 异常不破坏模型调用。
- OpenAI Responses 与 Anthropic Messages 的 JSON、SSE 文本和工具参数在本地恢复；
- SSE 占位符跨分片时使用增量状态恢复，避免把半个占位符发送给客户端。

## 非防御范围

- 本机被恶意软件控制；
- 管理员执行内存转储；
- 用户绕过代理；
- 图片、音频和附件中的隐私；
- WebSocket 与绕过当前 HTTP 接口的其他协议；
- 模型从公开信息推测真实身份；
- 上游自行生成新的敏感信息。

## 残余风险

- 规则可能漏报或误报；
- 匿名化可能影响必须读取原值的任务；
- 当前只恢复已支持 JSON 字段和 SSE 事件中的文本或工具参数；
- 姓名、自然语言地址和非结构化附件仍可能漏检；
- Provider 协议升级可能需要更新 Adapter。

> PrivacyTap 用于降低敏感信息外发和进入观测系统的概率，不是法律合规认证产品。
