# PrivacyTap 威胁模型

## 保护资产

- Prompt 中五类业务敏感信息；
- Prompt 中误输入的 API Key；
- 请求级占位符映射；
- 安全日志和 Langfuse Trace。

## 信任边界

本机 PrivacyTap 进程和调用客户端属于信任域；第三方模型、观测平台、日志备份和普通运维访问者不应获得原始敏感值。

## 防御范围

- 原始 PII 在请求离开本机前被替换；
- API Key 在请求上游之前被阻断；
- Vault 不序列化、不落盘、不上传；
- 日志回调只接收脱敏事件；
- 合法 Authorization Header 只转发，不记录；
- 并发请求使用独立 Vault。

## 非防御范围

- 本机被恶意软件控制；
- 管理员进行进程内存转储；
- 用户绕过代理直接调用模型；
- 图片、音频、文件附件中的隐私；
- 模型根据公开信息猜测真实身份；
- 上游自行生成用户未输入的新敏感信息。

## 残余风险

- 正则可能出现漏报或误报；
- 业务确实需要原值推理时，匿名化会降低可用性；
- 非 JSON 上游响应无法进行占位符恢复；
- 当前不支持流式返回。

> **声明：** PrivacyTap 降低敏感信息离开本机和进入观测系统的概率，但不是法律合规认证产品，也不防御已被攻陷的本机。
